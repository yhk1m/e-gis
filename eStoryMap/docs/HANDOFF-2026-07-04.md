## 2. 아키텍처 (핵심만)

- **Electron 3프로세스**: `electron/main.js`(+`fileService.js`)는 fs·다이얼로그·IPC만. `preload.js`가 `window.egisFS` 화이트리스트 노출(sandbox+contextIsolation). 렌더러는 순수 웹.

- **주요 모듈**: `egisParse`(.egis 검증) → `SourceRegistry`({src,layer}→OL 레이어) · `DemRenderer`(래스터) · `LocalStore`(.esm 직렬화+2초 디바운스 자동저장) · `AuthManager`(Supabase 인증, 주입식) · `CloudSync`(클라우드 테이블) · `StartScreen`(시작화면: 로컬 목록+클라우드 목록+로그인).

## 4. ⚠️ 함정 목록 (어기면 사고)

1. **vite.config.js에 vite-plugin-electron `renderer` 옵션·`optimizeDeps` 추가 금지** — 렌더러 전체가 조용히 죽는다(M2 geotiff 사고, 8c02e84).
2. **package.json에 `"type":"module"` 추가 금지** — electron main은 CJS 번들이어야 함.
3. **`signOut`은 반드시 `{scope:'local'}`** — 기본 global은 같은 계정의 e-GIS 웹 세션까지 무효화(supabase 프로젝트 공유).
4. **supabase 클라이언트는 AuthManager·CloudSync가 같은 인스턴스 공유** (`src/main.js`의 `const supabase = createSupabaseClient()` 하나).
5. **.egis `view.center`는 EPSG:4326 경위도** (마스터 스펙의 "3857" 표기는 오류로 정정됨). 카메라 복원 시 `fromLonLat`.
6. **eGIS 저장소는 다른 세션이 동시 작업할 수 있음** — 커밋 전 `git diff --cached --name-only`로 인덱스 오염 확인, `git add`는 항상 명시적 경로만 (2026-07-04 실제로 42파일 휩쓸림 사고 → reset으로 정정한 이력 있음).
7. Supabase URL/anon key는 하드코딩이 의도된 결정(공개 전제, 보안 경계는 RLS). env로 옮기지 말 것.
