# e-GIStory M4: 카메라 (캡처 + 페이지 전환 비행) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** "📷 이 위치로 캡처" 버튼으로 현재 지도 시점을 페이지에 저장하고, 페이지 전환 시 그 시점으로 부드럽게 비행(애니메이션)한다.

**Architecture:** 카메라는 M3 모델의 `page.camera`({center: [lon,lat] EPSG:4326, zoom} — .egis view와 동일 포맷)에 저장한다. `MapView.getCamera()`(현재 시점 읽기, e-GIS `MapManager.getCenter/getZoom` 이식)와 `StoryDoc.setPageCamera`(순수 변이)로 캡처하고, 새 모듈 `shared/CameraAnimator`(OL `view.animate` 래퍼, center+zoom 통합 800ms inAndOut)로 비행한다. **적용 시점이 핵심**: `refresh()`는 체크박스 토글마다 실행되므로 카메라는 refresh에 넣지 않고 **페이지 전환(onSelect)에서만** `flyTo`한다.

**Tech Stack:** 기존 그대로 — Electron 30 + Vite 5 + Vanilla JS + OL 9(`ol/easing` 추가 사용) + Vitest(jsdom). 신규 의존성 없음.

## 현재 코드 기준 확정 사실 (2026-07-03, 112 tests green, HEAD cf5a5b4)

1. **카메라 포맷 규약**: `page.camera = {center:[lon,lat](4326), zoom} | null`(StoryDoc `makePage`), .egis `view`와 동일(스펙 §2 "상호 변환 불필요"). e-GIS `MapManager.getCenter()`도 `toLonLat(view.getCenter())` 반환(4326) — MapManager.js:521-524 실측.
2. e-GIS의 이동 애니메이션(`setCenter/setZoom`, MapManager.js:538-564)은 **center/zoom 각각 300ms 개별** `view.animate` — 페이지 전환용으로는 통합 비행이 매끄러우므로 CameraAnimator는 center+zoom을 **한 번의 animate**로 묶는다(스펙 §3 "view.animate 래핑, easing 커스텀").
3. `src/core/MapView.js` — `setView(center4326, zoom)`은 `fromLonLat` 적용. import는 현재 `{ fromLonLat }`뿐(toLonLat 추가 필요). jsdom에서 MapView 생성 가능.
4. `src/core/StoryDoc.js` — `addPage`가 camera를 복사(M3), `setPageCamera`는 없음. `touch(doc)` 패턴.
5. `src/main.js`(M3 T9 상태) — `refresh()` 단일 갱신 지점(visibility+패널), `pageList` 핸들러 `onSelect(pageId){ currentPageId = pageId; refresh(); }`. `mapView.map`으로 OL Map 접근 가능. StoryDoc import는 `{ createStoryDoc, addSource, addPage, removePage, getPage, setLayerVisible, nextSourceId }`.
6. `src/core/StoryMapRenderer.js` — 함수 끝에 `// (M4) page.camera 적용 지점 — CameraAnimator에서 이동/애니메이션 담당 예정.` 주석 존재(이번에 결정 반영해 갱신).
7. `index.html` — `#workspace` 안에 `<aside id="sidebar">…</aside>` 와 `<div id="map"></div>`. OL이 `#map` 안에 viewport를 만들므로 캡처 버튼은 `#map`과 **형제**로 두고 공통 부모(`#map-stage`)에 absolute 배치한다.
8. OL API: `view.animate({center, zoom, duration, easing})`, `view.cancelAnimations()`(연타 시 이전 비행 취소), `ol/easing`의 `inAndOut`. jsdom에서는 실제 애니메이션 진행이 불가하므로 **CameraAnimator 테스트는 `vi.spyOn(view, 'animate').mockImplementation(()=>{})`로 호출 계약만 검증**(실비행은 수동 스모크).
9. ⚠️ vite.config.js에 `renderer` 플러그인 금지(기존 함정), `npm run dev`는 사람 스모크 전용.

// © 2026 김용현
import { describe, it, expect, vi } from 'vitest';
import View from 'ol/View';
import { fromLonLat } from 'ol/proj';
import { CameraAnimator } from './CameraAnimator.js';

function setup() {
  const view = new View({ center: [0, 0], zoom: 2 });
  const animate = vi.spyOn(view, 'animate').mockImplementation(() => {});
  const cancel = vi.spyOn(view, 'cancelAnimations');
  return { animate, cancel, animator: new CameraAnimator(view) };
}

describe('CameraAnimator.flyTo', () => {
  it('이전 비행을 취소하고 center(3857 변환)+zoom을 한 번에 애니메이트한다', () => {
    const { animate, cancel, animator } = setup();
    animator.flyTo({ center: [129.05, 35.15], zoom: 11 });
    expect(cancel).toHaveBeenCalled();
    expect(cancel.mock.invocationCallOrder[0]).toBeLessThan(animate.mock.invocationCallOrder[0]);
    const args = animate.mock.calls[0][0];
    expect(args.center).toEqual(fromLonLat([129.05, 35.15]));
    expect(args.zoom).toBe(11);
    expect(args.duration).toBe(800);
    expect(typeof args.easing).toBe('function');
  });

  it('duration 옵션이 반영된다', () => {
    const { animate, animator } = setup();
    animator.flyTo({ center: [127, 37], zoom: 9 }, { duration: 300 });
    expect(animate.mock.calls[0][0].duration).toBe(300);
  });

  it('camera가 null이면 아무것도 하지 않는다', () => {
    const { animate, cancel, animator } = setup();
    animator.flyTo(null);
    expect(animate).not.toHaveBeenCalled();
    expect(cancel).not.toHaveBeenCalled();
  });

  it('center가 배열이 아니면 무시한다(손상 카메라 가드)', () => {
    const { animate, animator } = setup();
    animator.flyTo({ center: 'busan', zoom: 9 });

  });

```js
// © 2026 김용현
// eStoryMap/src/shared/CameraAnimator.js
// 페이지 카메라 비행 — OL view.animate 얇은 래퍼(상위 스펙 §3 shared/CameraAnimator).
// e-GIS MapManager.setCenter/setZoom은 300ms 개별 애니메이션이지만, 페이지 전환은
// center+zoom을 한 번의 800ms inAndOut 비행으로 묶는다(슬라이드 전환감).
import { fromLonLat } from 'ol/proj';
import { inAndOut } from 'ol/easing';

export class CameraAnimator {
  /** @param {import('ol/View').default} view - OL View (mapView.map.getView()) */
  constructor(view) {
    this.view = view;
  }

  /**
   * 카메라로 비행. 연속 전환(연타) 시 이전 비행은 취소된다.
   * @param {{center:number[], zoom:number}|null} camera - center는 EPSG:4326 [경도, 위도]
   * @param {{duration?:number}} [options]
   */
  flyTo(camera, { duration = 800 } = {}) {
    if (!camera || !Array.isArray(camera.center)) return;
    this.view.cancelAnimations();
    this.view.animate({
      center: fromLonLat(camera.center),
      zoom: camera.zoom,
      duration,
      easing: inAndOut,
    });
  }
}
```
