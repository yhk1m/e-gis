# e-GIS · e-GIStory 사용 설명서

교육용 GIS 도구 모음의 전체 기능을 기능별로 설명합니다. 두 프로그램은 서로 연동됩니다.

- **e-GIS** — 웹 브라우저에서 바로 쓰는 GIS 편집·분석 도구입니다. 주소는 `https://e-gis.kr`, 설치가 필요 없습니다.
- **e-GIStory** — e-GIS로 만든 지도를 슬라이드로 엮어 **스토리맵(발표 자료)**을 만드는 데스크톱 프로그램입니다. e-GIS 웹앱을 안에 품고 있어서 지도 작업과 발표를 한 창에서 병행할 수 있습니다.

> 두 프로그램은 **`.egis` 프로젝트 파일**로 이어집니다. e-GIS에서 지도를 만들어 `.egis`로 저장 → e-GIStory에서 그 파일을 소스로 불러와 발표를 구성하는 흐름입니다. 자세한 연동은 [3부](#3부-두-프로그램-연동하기)를 보세요.

# 1부. e-GIS (웹 GIS)

## 1-1. 시작하기와 화면 구성

e-GIS는 설치 없이 웹 브라우저에서 바로 실행됩니다. 접속하면 대한민국 지도가 기본으로 표시됩니다.

화면은 다음 네 영역으로 이루어집니다.

| 영역 | 위치 | 역할 |
|---|---|---|
| **메뉴 바** | 상단 | 프로젝트·편집·보기·레이어·측정·분석·주제도 등 모든 기능의 진입점 |
| **도구 모음(툴바)** | 메뉴 바 아래 | 확대/축소, 선택, 그리기, 측정, 위치 검색 등 자주 쓰는 도구 |
| **사이드 패널** | 왼쪽 | `레이어` 탭(레이어 목록)과 `브라우저` 탭(파일 불러오기) |
| **상태 표시줄** | 하단 | 현재 좌표, 축척, 좌표계, 조회수 |

- 오른쪽 위 **❓ 버튼**으로 앱 안에 내장된 간단 설명서를 열 수 있습니다.
- 오른쪽 위 **테마 전환 버튼**(해/달 아이콘)으로 밝은/어두운 테마를 바꿉니다.
- 사이드 패널은 패널 옆 **접기/펴기 화살표**로 숨기거나 다시 펼 수 있습니다.
- 메뉴 바가 좁은 창에서는 글자가 사라지고 **아이콘만** 표시됩니다(마우스를 올리면 이름 툴팁이 뜹니다).

## 1-2. 데이터 불러오기

e-GIS에 데이터를 올리는 방법은 세 가지입니다.

### 내장 데이터 불러오기 (가장 쉬움)

메뉴 바의 **📂 데이터 불러오기**를 누르면, 미리 준비된 한국·세계 데이터를 바로 지도에 올릴 수 있습니다. 세 종류가 있습니다.

- **공간 데이터** — 지도 경계 그 자체. `대한민국 시군구`, `대한민국 시도`(행정구역 개편 전/후 두 종류), `세계 국가`.
- **속성 데이터** — 지도에 결합해 쓰는 통계 표. `시도별 인구`, `시도별 GRDP`(지역내총생산). 개편 전/후 버전이 있습니다. → [테이블 결합](#테이블-결합)으로 공간 데이터에 붙여 씁니다.
- **래스터(고도) 데이터** — 전국 시군구별 **해발고도(DEM)** 234종. 지형음영·경사도·등고선 분석의 재료입니다.

### 파일 가져오기

내 컴퓨터의 GIS 파일을 올립니다. 두 가지 방법이 있습니다.

- **방법 1** — 왼쪽 `레이어` 패널의 **+ 버튼** → 파일 선택
- **방법 2** — 왼쪽 `브라우저` 탭에서 파일을 **드래그 앤 드롭**

지원 형식:

| 형식 | 확장자 | 비고 |
|---|---|---|
| GeoJSON | `.geojson`, `.json` | 가장 널리 쓰이는 벡터 형식 |
| Shapefile | `.zip` | `.shp/.dbf/.shx` 등을 하나의 zip으로 압축해 올립니다 |
| GeoPackage | `.gpkg` | |
| DEM(고도) | `.tif`, `.tiff`, `.img` | 래스터 분석용 |

### 좌표 데이터 가져오기

위도·경도가 담긴 표에서 점 레이어를 만듭니다.

**메뉴 → 레이어 → 좌표 데이터 가져오기** → `.csv/.txt/.xlsx/.xls` 파일 선택 → 위도·경도가 든 열을 지정하면 점 레이어가 생성됩니다.

## 1-3. 지도 조작 · 배경지도 · 좌표계

### 지도 이동과 확대/축소

- **이동** — 마우스로 드래그하거나 방향키
- **확대/축소** — 마우스 휠, 또는 툴바의 **＋ / －** 버튼
- **전체 범위** — 툴바의 **지구본 버튼**(모든 레이어가 한눈에 들어오도록 맞춤)
- **위치 검색** — 툴바 검색창에 주소·장소명을 입력해 해당 위치로 이동

### 배경지도

지도 위 오른쪽의 **배경지도 버튼**으로 바탕 지도를 고릅니다.

- **일반지도** — OpenStreetMap 계열의 밝은/어두운 지도(테마에 맞춰 자동)
- **위성** — 위성 영상
- **위성 + 라벨** — 위성 영상 위에 지명·경계 라벨

이 밖에 지도 위에는 **나침반**(북쪽 방향 회전 조절), **현재 위치** 버튼이 있습니다.

### 좌표계

상태 표시줄 오른쪽의 **좌표계 표시**를 클릭하면 좌표 표시용 좌표계를 바꿀 수 있습니다.

| 코드 | 이름 |
|---|---|
| EPSG:4326 | WGS 84 (경위도) — 기본값 |
| EPSG:3857 | 웹 메르카토르 |
| EPSG:5179 | Korea 2000 / UTM-K |
| EPSG:5186 | Korea 2000 / 중부원점 |
| EPSG:5187 | Korea 2000 / 동부원점 |
| EPSG:5188 | Korea 2000 / 서부원점 |

바꾸면 상태 표시줄의 좌표 표시가 해당 좌표계 값으로 바뀝니다.

## 1-4. 레이어 관리

왼쪽 `레이어` 패널에서 지도에 올라간 레이어들을 관리합니다.

- **표시/숨기기** — 레이어 이름 왼쪽의 **눈 아이콘** 클릭. 맨 위 헤더의 체크박스로 전체를 한 번에 켜고 끕니다.
- **선택** — 레이어 이름 클릭. `Shift+클릭`으로 여러 개를 한꺼번에 선택합니다. (버퍼·주제도 등 상당수 기능은 **먼저 대상 레이어를 선택**해야 실행됩니다.)
- **순서 변경** — 레이어를 위아래로 **드래그**. 위에 있을수록 지도에서 위에 그려집니다.
- **이름 변경** — ⋮ 메뉴 → 이름 변경(또는 메뉴 → 레이어 → 이름 변경)
- **삭제** — 레이어 선택 후 `Delete` 키, 또는 ⋮ 메뉴 → 삭제

### 레이어 ⋮ (점 세 개) 메뉴

각 레이어 행의 **⋮ 버튼**을 누르면 그 레이어에만 적용되는 작업이 나옵니다.

- **레이어로 이동** — 그 레이어가 꽉 차게 지도 이동
- **이름 변경**
- **레이어 복사** — 도형·속성·스타일·주제도 설정까지 그대로 복제해 `(사본)`으로 추가. 같은 데이터를 여러 방식으로 시각화할 때 유용합니다. *(벡터 레이어만)*
- **내보내기 (GeoJSON)** — 이 레이어만 GeoJSON 파일로 저장 *(벡터 레이어만)*
- **속성 테이블** — 이 레이어의 표 열기
- **색상 변경** — 채우기/테두리 색, 불투명도, 선 두께, 파선, 점 크기 지정
- **삭제**

### 레이어 스타일(색상) 지정

⋮ 메뉴 → **색상 변경**에서 세밀하게 꾸밉니다.

- **채우기 색** / **테두리 색**을 따로 지정
- **채우기 불투명도** / **테두리 불투명도**
- **선 두께**, **파선**(실선/대시/점선/일점쇄선)
- 점 레이어는 **점 크기**

> 여기서 지정한 스타일은 `.egis`로 저장하면 함께 저장되어, e-GIStory로 불러올 때도 그대로 표현됩니다.

## 1-5. 도형 그리기

툴바의 그리기 도구로 지도에 직접 도형을 그립니다. 그린 도형은 새 레이어(또는 선택한 레이어)에 피처로 추가됩니다.

| 도구 | 그리는 법 |
|---|---|
| **점** | 지도에서 클릭 |
| **선** | 클릭으로 꼭짓점 추가, **더블클릭으로 완료** |
| **면(폴리곤)** | 클릭으로 꼭짓점 추가, **더블클릭으로 완료** |
| **멀티포인트** | 여러 점을 클릭 → 다시 도구 클릭(또는 더블클릭)으로 하나의 피처로 저장 |
| **멀티라인** | 여러 선을 그리고 각각 더블클릭으로 완료 |
| **멀티폴리곤** | 여러 폴리곤을 그리고 각각 더블클릭으로 완료 |

- **이미지 업로드** — 툴바의 이미지 도구로 PNG/JPG/SVG를 지도에 얹을 수 있습니다(정확한 위치 맞춤은 [지리참조](#지리참조-georeferencing) 사용).

## 1-6. 피처 선택과 편집

### 선택

- 툴바의 **선택 도구**를 켜고 피처를 클릭
- **드래그**로 범위를 그려 여러 피처를 한 번에 선택
- `Shift+클릭` 다중 선택 · `Ctrl+A` 전체 선택
- 선택을 풀려면 툴바의 **선택 취소** 버튼(선택 중에만 나타남)

### 편집

- **꼭짓점 수정** — 피처 선택 후 꼭짓점을 드래그
- **피처 합치기** — 여러 피처를 선택한 뒤 툴바의 **합치기** 버튼(박음질하듯 하나로 병합). 메뉴 → 편집 → 피처 합치기도 동일
- **피처 자르기** — 자르기 도구로 선을 그어 하나의 피처를 둘로 분할
- **삭제** — 선택 후 `Delete` 키, 또는 툴바의 삭제 버튼
- **복사/붙여넣기** — `Ctrl+C` / `Ctrl+V`

## 1-7. 속성 테이블 · 테이블 결합 · 필드 계산기 · 라벨

### 속성 테이블

레이어의 데이터를 표로 봅니다. 레이어 선택 → **메뉴 → 레이어 → 속성 테이블** (또는 ⋮ 메뉴 → 속성 테이블).

- **행 선택** — 행 클릭. `Shift+클릭` 범위 선택, `Ctrl+클릭` 개별 추가. 헤더 체크박스로 전체 선택
- **셀 편집** — 셀을 더블클릭해 값 입력 후 `Enter`
- **지도에서 보기** — 행 선택 후 버튼을 누르면 해당 피처로 지도 이동
- **피처 삭제** — 행 선택 후 삭제 버튼

### 테이블 결합

통계 표(CSV/엑셀)를 공간 레이어에 **공통 키**(예: 행정구역 코드/이름)로 붙입니다.

레이어 선택 → **메뉴 → 레이어 → 테이블 결합** → `.csv/.txt/.xlsx/.xls` 선택 → 레이어의 열과 표의 열 중 **일치시킬 키 열**을 지정 → 결합. 결합 후에는 표의 값이 레이어 속성에 추가되어 [단계구분도](#단계구분도) 등에 쓸 수 있습니다.

> 예: `대한민국 시도`(공간) + `시도별 인구`(속성)를 시도명으로 결합 → 인구 단계구분도 제작.

### 필드 계산기

기존 속성으로 **새 열을 계산**합니다. 레이어 선택 → **메뉴 → 레이어 → 필드 계산기** → 새 필드 이름과 계산식을 입력(예: `인구밀도 = 인구 / 면적`).

### 라벨

피처 위에 속성값을 글자로 표시합니다. 레이어 선택 → **메뉴 → 레이어 → 라벨 설정** → 표시할 필드와 글꼴·크기 등을 지정.

## 1-8. 측정

- **거리 측정** — 메뉴 → 측정 → 거리 측정 → 지도에서 클릭해 경로를 그림
- **면적 측정** — 메뉴 → 측정 → 면적 측정 → 폴리곤을 그림
- **측정 결과 지우기** — 메뉴 → 측정 → 측정 결과 지우기

## 1-9. 벡터 분석

**메뉴 → 벡터 분석**에 모여 있습니다.

### 버퍼 분석

선택한 레이어의 피처 주변에 일정 거리의 영역을 만듭니다. 레이어 선택 → 버퍼 분석 → **거리와 단위** 입력. **버퍼 병합(Dissolve)**을 켜면 겹치는 버퍼를 하나로 합칩니다.

### 공간 연산

두 레이어 사이의 집합 연산입니다.

- **교집합(Intersect)** — 두 레이어가 겹치는 영역만
- **합집합(Union)** — 두 레이어를 합친 영역
- **차집합(Difference)** — 한 레이어에서 다른 레이어를 뺀 영역
- **클리핑(Clip)** — 한 레이어를 다른 레이어 경계로 오려내기

### 등시선 분석

한 지점에서 **일정 시간 안에 갈 수 있는 범위**를 그립니다. 이동 수단은 **자동차 · 자전거 · 도보** 중 선택합니다.

### 최단경로 분석

두 지점(및 경유지) 사이의 최단 경로를 찾습니다. 이동 수단을 고르고 **+ 경유지 추가**로 중간 지점을 넣을 수 있습니다.

> **등시선·최단경로는 OpenRouteService의 무료 API 키가 필요합니다.** 처음 사용 시 키 입력란이 나오며, 입력한 키는 브라우저에 저장되어 다음부터 자동 사용됩니다. (openrouteservice.org에서 무료 발급)

---

## 1-10. 래스터 분석

DEM(고도) 레이어를 이용한 지형 분석입니다. **메뉴 → 래스터 분석**. 재료가 되는 고도 데이터는 [📂 데이터 불러오기](#내장-데이터-불러오기-가장-쉬움)의 래스터에서 시군구별로 가져올 수 있습니다.

- **해발고도 (지형음영)** — 고도를 색으로 표현하고 음영을 넣어 지형의 높낮이를 시각화
- **경사도 (Slope)** — 지형의 기울기 정도
- **경사방향 (Aspect)** — 사면이 향하는 방향
- **래스터 계산기 (값 필터)** — 특정 고도값 범위만 남기는 등 조건 필터
- **등고선 생성** — 같은 고도를 잇는 등고선을 벡터 선으로 생성(간격 지정). 주곡선/계곡선 굵기를 다르게 표현
- **지리참조 (Georeferencing)** — 아래 설명

### 지리참조 (Georeferencing)

종이 지도·항공사진 같은 **좌표가 없는 이미지**를 실제 지도 위에 정확히 맞춰 얹습니다.

1. 메뉴 → 래스터 분석 → 지리참조 → PNG/JPG/SVG 이미지 선택
2. 왼쪽 이미지 창에서 한 점을 찍고, 지도에서 그에 대응하는 실제 위치를 찍습니다 — 이 **기준점(GCP)**을 여러 개 만듭니다
3. 기준점 3개 이상이면 **아핀 변환**, 4개 이상이면 **원근(호모그래피) 변환**으로 이미지가 지도에 맞게 휘어져 표시됩니다
4. 이미지 창은 휠로 확대해 정밀하게 찍을 수 있습니다

> 지리참조 결과는 `.egis`로 저장하면 이미지·기준점·변환 방식까지 함께 저장되어 복원됩니다.

## 1-11. 주제도 (시각화)

수치 데이터를 지도에 시각적으로 표현합니다. **메뉴 → 주제도**. 대부분 **먼저 레이어를 선택**해야 실행됩니다.

### 단계구분도

수치 속성값을 **색의 진하기**로 표현합니다(예: 인구가 많을수록 진한 색). 속성·분류 방식·색상표를 고르면 계급별로 채색되고, 지도 위에 범례가 생깁니다.

- 지도 위 **범례 제목을 클릭해 직접 수정**할 수 있고, 레이어 이름과 양방향으로 동기화됩니다.
- 단위·숫자 표기 형식·반올림 자리를 지정할 수 있습니다.

### 도형표현도

각 지역 위에 **파이/막대 차트**를 얹어 여러 값을 한 번에 비교합니다.

- **차트 유형** — 파이 차트 · 막대 차트 · 100% 막대 차트
- **표시할 필드** — 차트에 넣을 숫자 열들을 체크(막대는 1개 이상, 파이·100%막대는 2개 이상)
- **필드별 색 지정** — 각 필드 옆 색 상자를 클릭해 원하는 색으로 직접 설정
- **수치 라벨 표시** — 켜면 파이·100%막대는 **%**, 막대 차트는 **값**을 도형 위에 표시
- **크기 기준 필드** — 특정 값에 비례해 차트 크기를 다르게(예: 총인구가 클수록 큰 차트)
- **최소/최대 크기** 슬라이더

> 도형표현도는 차트가 지도에 그려지므로 `.egis` 저장·이미지 내보내기·스토리맵 발표·PDF에 모두 그대로 나옵니다.

### 히트맵

점 데이터의 **밀도**를 열지도로 표현합니다(점이 몰린 곳일수록 붉게).

### 카토그램

수치값에 비례해 **지역 면적 자체를 왜곡**해 표현합니다(예: 인구가 많은 지역이 크게). 라벨 표시를 켤 수 있습니다.

## 1-12. 저장과 내보내기

### 프로젝트 저장 (.egis)

지금까지의 모든 레이어·스타일·주제도·지도 위치를 하나의 파일로 저장합니다. **메뉴 → 프로젝트 → 저장** → 이름 입력. `.egis` 파일이 만들어집니다.

- `.egis`에는 **벡터 도형·속성, 세밀한 스타일(채우기/테두리 색 등), 단계구분도·도형표현도·카토그램 설정, 래스터(고도·지리참조) 데이터, 지도 뷰**가 모두 담깁니다.
- 다시 열려면 **메뉴 → 프로젝트 → 열기**.

> ⚠️ 스타일·주제도가 파일에 포함되려면 **저장 시점의 상태**가 기록됩니다. 예전에 저장한 파일에 색이 없다면, e-GIS에서 한 번 열어 **다시 저장**하면 새 형식으로 갱신됩니다.

### 레이어 내보내기

특정 레이어만 표준 형식으로 내보냅니다. 레이어 선택 → **메뉴 → 레이어 → 레이어 내보내기** → **GeoJSON** 또는 **Shapefile**. (레이어 ⋮ 메뉴의 "내보내기"는 GeoJSON.)

### 지도 이미지 내보내기

현재 지도 화면을 이미지로 저장합니다. **메뉴 → 프로젝트 → 지도 내보내기**. 제목·나침반·텍스트 상자 등을 얹어 내보낼 수 있습니다.

# 2부. e-GIStory (스토리맵 데스크톱)

e-GIStory는 e-GIS로 만든 지도를 **슬라이드로 엮어 발표 자료(스토리맵)**를 만드는 데스크톱 프로그램입니다. 창 맨 위에는 세 개의 탭이 있습니다.

- **e-GIS** — 웹 GIS를 앱 안에서 그대로 사용 (1부 내용 전부)
- **e-GIStory** — 스토리맵 편집기 (2부 내용)
- **⬓ 분할** — e-GIS와 e-GIStory를 나란히 (→ [2-13](#2-13-분할-모드--e-gis와-나란히))

## 2-1. 시작하기 — 프로젝트 만들기·열기

프로그램을 켜면 **시작 화면**이 나옵니다.

- **새 스토리맵 만들기** — 제목을 입력하고 만들면 빈 프로젝트가 생성됩니다. 프로젝트는 `.esm` 파일로 자동 저장됩니다(저장 폴더는 툴바의 `저장 폴더 열기`로 확인).
- **기존 프로젝트 열기** — 목록에서 프로젝트를 클릭
- **클라우드 프로젝트 열기** — 로그인하면 클라우드에 저장한 스토리맵 목록이 함께 표시됩니다
- **로그인/로그아웃** — 시작 화면에서 이메일·비밀번호로 로그인(회원가입은 e-gis.kr에서)

> 편집 중에는 **2초 간격으로 자동 저장**되므로 따로 저장 버튼을 누를 필요가 없습니다.

---

## 2-2. 화면 구성

편집기는 세로로 세 칸입니다.

| 영역 | 위치 | 역할 |
|---|---|---|

| **슬라이드 캔버스** | 가운데 | 16:9 비율의 지도. 발표에서 보일 그대로 미리 보임 |
| **콘텐츠 패널** | 오른쪽 | 현재 슬라이드의 종류·글·배경 등 편집 |

상단 툴바의 주요 버튼:

| 버튼 | 기능 |

| .egis 열기 | e-GIS 지도 파일을 소스로 불러오기 |
| 저장 폴더 열기 | `.esm` 파일이 저장된 폴더 열기 |
| ▶ 발표 | 발표 모드 시작 |
| 📊 발표 PDF | 발표 슬라이드를 PDF로 저장 |
| 🌐 게시 | 스토리맵을 웹 링크로 공개 |
| 레이아웃 / 위치 / 글꼴 | 발표 텍스트 배치와 글꼴 (프로젝트 전체) |
| 범례 | 지도 위 범례 표시 여부 |

## 2-3. 소스와 레이어

스토리맵의 지도는 e-GIS에서 만든 **`.egis` 파일을 소스로 불러와** 구성합니다.

- **소스 추가** — 툴바 **`.egis 열기`** → 파일 선택. 그 파일 안의 레이어들이 소스로 등록됩니다.
- **레이어 켜기/끄기** — 왼쪽 `소스` 패널에서 각 레이어의 **체크박스**로 현재 슬라이드에 보일 레이어를 고릅니다. **슬라이드마다 다르게** 설정할 수 있습니다(1번 슬라이드엔 인구, 2번 슬라이드엔 고도 식).
- **전체 선택 / 전체 해제** — 현재 슬라이드의 모든 레이어를 한 번에
- **소스 제거** — 소스 이름 옆 **✕ 버튼**. 모든 슬라이드에서 그 소스의 레이어가 사라집니다.

> 핵심: 레이어의 **켜짐/꺼짐은 슬라이드별로 기억**됩니다. 같은 지도를 두고 슬라이드를 넘길 때마다 다른 레이어를 보여주며 이야기를 전개할 수 있습니다.

---

## 2-4. 페이지(슬라이드) 관리

왼쪽 `페이지` 패널에서 슬라이드를 관리합니다.

- **추가** — `+ 페이지` 버튼. **현재 슬라이드를 복제**해 추가되므로(레이어·위치 포함) 조금씩 바꿔가며 만들기 좋습니다.
- **순서 변경** — 슬라이드를 **드래그**
- **이름 변경** — 슬라이드의 ✏️ 버튼(또는 이름 더블클릭)
- **삭제** — 슬라이드의 삭제 버튼(확인 후 삭제, 최소 1개는 유지)

---

## 2-5. 콘텐츠 편집

오른쪽 `콘텐츠` 패널에서 현재 슬라이드의 내용을 만듭니다.

### 슬라이드 종류

맨 위 **종류** 선택에 따라 슬라이드의 성격이 달라집니다.

| 종류 | 화면 | 용도 |
|---|---|---|
| **지도** | 지도 + 글 카드 | 기본. 지도를 배경으로 설명을 얹음 |
| **제목(표지)** | 지도 없이 큰 제목 | 챕터 표지 |
| **사진/영상** | 지도 없이 미디어 | 사진·유튜브 등을 크게 |
| **글** | 지도 없이 글만 | 텍스트 슬라이드 |

### 글 입력과 마크다운

- **제목 / 본문 / 캡션**을 입력합니다(종류에 따라 라벨이 조금씩 다름).
- 본문은 **마크다운**을 지원합니다: `# 제목`, `**굵게**`, `- 목록` 등. 툴바의 서식 버튼과 `? 사용법` 도움말도 있습니다.

### 미디어 임베드 (사진/영상)

`사진/영상` 종류에서 본문에 링크를 **한 줄로** 붙여넣으면 자동으로 삽입됩니다.

- **YouTube 링크** → 재생 가능한 영상
- **구글 드라이브 이미지 링크** → 이미지 (드라이브 파일은 "링크가 있는 모든 사용자" 공개여야 함)
- **일반 이미지 URL**(`.jpg/.png/...`) → 이미지

### 사진 정렬과 2단 레이아웃

- **사진 정렬** — 좌 / 가운데 / 우
- **2단 레이아웃** — 켜면 한쪽에 사진, 다른 쪽에 글을 나란히. **칸 너비 비율**을 슬라이더로 조절

---

## 2-6. 지도 위치(카메라) 다루기

슬라이드마다 **지도의 위치·확대 정도(카메라)**를 따로 저장합니다. 발표에서 슬라이드를 넘기면 지도가 그 위치로 부드럽게 이동합니다.

지도 아래 버튼:

- **📷 이 위치로 캡처** — 지금 보이는 지도 화면을 **이 슬라이드의 카메라**로 저장
- **📌 모든 슬라이드에 이 위치** — 현재 지도 위치를 **모든 슬라이드에** 한 번에 적용
- **🔗 위치 가져오기** — **다른 슬라이드의 위치**를 현재 슬라이드로 복사

---

## 2-7. 슬라이드 꾸미기 — 배경지도·배경색·글꼴

### 배경지도 (슬라이드별)

콘텐츠 패널에서 슬라이드마다 지도 배경을 고릅니다: **일반 / 위성 / 위성 + 라벨**.

### 배경색

- **슬라이드 배경색** — 콘텐츠 패널에서 현재 슬라이드의 배경/글자색
- **모든 슬라이드에 적용** — 이 색을 프로젝트 전체 기본으로

### 글꼴 (프로젝트 전체)

툴바의 **글꼴** 선택: **맑은 고딕 / Noto Sans / Noto Serif / 시스템 글꼴**(설치된 글꼴 이름 직접 입력). 슬라이드·발표·PDF·게시본 전체에 적용됩니다.

---

## 2-8. 발표 레이아웃과 범례

### 레이아웃과 위치 (프로젝트 전체)

지도 슬라이드에서 **글 카드의 모양과 위치**를 정합니다.

- **레이아웃** — 밴드(가장자리 그라데이션 띠) / 패널(단색 칸) / 카드(떠 있는 카드)
- **위치** — 왼쪽 / 오른쪽 / 위 / 아래

### 범례

툴바의 **범례** 체크박스로 지도 위 범례 표시를 켜고 끕니다. 범례는 **드래그로 위치**를 옮기고, 항목 **라벨을 편집**하거나 개별 항목을 **숨길** 수 있습니다(편집기에서). 발표에서는 이 설정 그대로 정적으로 표시됩니다.

---

## 2-9. 발표 모드

툴바의 **▶ 발표**를 누르면 전체화면 발표가 시작됩니다.

- **이동** — 화면의 **◂ ▸ 화살표**, 또는 키보드 `←` `→` `Space` `PageUp/Down`, 처음/끝은 `Home` `End`
- **슬라이드 전환** — 지도 카메라가 부드럽게 이동, 글 카드가 레이아웃대로 표시
- **화살표 자동 숨김** — 마우스·키보드가 **2초간 없으면** 화살표가 사라지고, 움직이면 다시 나타납니다
- **사진 확대** — 발표 중 사진을 클릭하면 크게(라이트박스)
- **종료** — `Esc` 또는 ✕

실제 지도가 그대로 표시되므로 발표 중에도 **지도를 확대·이동**하며 설명할 수 있습니다.

---

## 2-10. 발표 PDF

툴바의 **📊 발표 PDF**로 발표 슬라이드를 **16:9 PDF**로 저장합니다. 지도는 고해상도로 캡처되며, 도형표현도 차트·범례·글이 모두 담깁니다. 배포용 유인물이나 오프라인 발표에 유용합니다.

---

## 2-11. 웹에 게시하기 (공유 링크)

스토리맵을 **누구나 볼 수 있는 웹 링크**로 공개합니다. (로그인 필요)

1. 툴바의 **🌐 게시** 클릭
2. 첫 게시면 확인 후 링크가 만들어집니다 — 주소는 **`https://e-gis.kr/{내 아이디}/{번호}`** 형태입니다. 아이디는 이메일에서 자동으로 만들어집니다(예: `fkv777@gmail.com` → `fkv777gmail`).
3. **링크 복사** / **브라우저에서 열기**로 공유

게시 후 대화상자에서:

- **재게시** — 수정한 내용으로 링크 내용을 갱신(스냅샷 방식이라, 재게시 전까지 방문자에게는 이전 내용이 보입니다)
- **링크 복사** / **브라우저에서 열기**
- **게시 취소** — 링크를 닫음(더 이상 열리지 않음)

방문자는 로그인 없이 **발표 모드 그대로** 스토리맵을 감상합니다(지도 조작·슬라이드 이동 가능).

> 게시는 [클라우드 동기화](#2-12-클라우드-동기화)(비공개 백업)와는 **별개**입니다. 게시 = 공개 스냅샷, 동기화 = 내 작업본 백업.

---

## 2-12. 클라우드 동기화

툴바의 **클라우드 동기화** 체크박스를 켜면(로그인 필요), 저장할 때마다 스토리맵이 **내 클라우드에도 자동 백업**됩니다. 다른 기기에서 로그인하면 시작 화면의 **클라우드 프로젝트 열기**로 이어서 작업할 수 있습니다. (공개가 아닌 개인 백업입니다.)

---

## 2-13. 분할 모드 — e-GIS와 나란히

**e-GIStory와 e-GIS를 한 화면에 나란히** 놓고 작업하거나 발표할 수 있습니다. 수업에서 스토리맵 발표를 띄워 두고 옆에서 e-GIS를 실시간 시연하는 데 유용합니다.

- **켜기/끄기** — 상단 **⬓ 분할** 탭. 켜면 한쪽에 e-GIS, 다른 쪽에 e-GIStory가 나란히 표시됩니다(기본: 왼쪽 e-GIS · 오른쪽 e-GIStory).
- **좌우 전환** — **⇄** 버튼으로 e-GIS를 왼쪽/오른쪽 어디에 둘지 바꿈
- **비율 조절** — 가운데 **구분선을 드래그**
- **한쪽 가득 채우기** — 구분선을 **화면 끝까지 끌면** 그쪽이 접히고 반대쪽이 가득 찹니다. 가장자리에 남은 구분선을 다시 끌면 분할로 복귀
- **분할 발표** — 분할 상태에서 **▶ 발표**를 누르면 분할 배치를 유지한 채 전체화면으로 발표가 진행됩니다. **반대쪽 e-GIS는 계속 조작 가능**합니다.

> 분할·좌우·비율 설정은 기억되어 다음에 켤 때 복원됩니다.

# 3부. 두 프로그램 연동하기

e-GIS와 e-GIStory는 **`.egis` 파일**로 이어집니다. 전형적인 흐름:

1. **e-GIS에서 지도 제작** — 데이터를 불러오고, 스타일·주제도(단계구분도·도형표현도 등)를 지정합니다.
2. **`.egis`로 저장** — 메뉴 → 프로젝트 → 저장. 도형·속성·스타일·주제도·래스터·지도 위치가 모두 담깁니다.
3. **e-GIStory에서 소스로 불러오기** — `.egis 열기`로 그 파일을 소스로 추가합니다. e-GIS에서 지정한 색·주제도가 그대로 표현됩니다.
4. **슬라이드로 구성** — 슬라이드마다 보일 레이어와 지도 위치를 정하고 글을 붙입니다.
5. **발표 / PDF / 웹 게시**로 공유합니다.

**주의할 점**

- `.egis`에 스타일·주제도가 담기려면 **e-GIS에서 저장한 시점의 상태**가 기록됩니다. 예전 파일에 색·차트가 없다면 e-GIS에서 한 번 열어 **다시 저장**하세요.
- **도형표현도**는 차트가 지도에 그려지므로 스토리맵 발표·PDF에 그대로 나옵니다.
- e-GIStory의 **분할 모드**를 쓰면 두 프로그램을 오가지 않고 한 화면에서 지도 수정과 발표 구성을 병행할 수 있습니다.
